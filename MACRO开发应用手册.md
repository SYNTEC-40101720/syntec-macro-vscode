# **前言**

为增加控制器应用弹性，新代控制器提供MACRO程序编辑功能。

当加工程序被宣告成MACRO格式后，该档案将如同一般程序语言，可使用特定数学函数，如此一来，除了原本即有的移动和补偿指令功能，更拥有逻辑判断及数学演算功能。

# **档案格式**

程序内容第一行需使用"%"宣告为标题行，并加入关键字"@MACRO"，才会被视为 MACRO 格式档处理。否则该档案将被视为ISO格式档处理。

ISO 格式档与 MACRO 格式档的程序内容解读差异如下

**ISO格式档**

**MACRO格式档**

不支援 MACRO 语法使用  
(有部分 MACRO 语法无法被正确解读，可能会跳出"COM-003 控制器解译宏程序时发现程序句法有误"的警报)

可以使用 MACRO 语法的完整功能。

每一行结尾加或不加分号";"皆可。

1\. 除了特定语法外(请参考语法说明内容)，每一行结尾都要加上分号";"。

2\. 10.120.45 前，如未加分号，会和下一行合并处理。若没有检查出警报，仍会正常执行。  
3\. 10.120.45 后，除了特定语法外，该行没有加上分号时，会跳出 COM-008【子句中没有结束的符号' ; '】。  
4\. 10.120.61 后，如未加分号，会和下一行合并处理。若没有检查出警报，仍会正常执行。

范例：

| **原加工程序** | **等效程序** | **说明** |
| --- | --- | --- |
| %@MACRO  <br>#1= SIN(100) // 漏加";"  <br>G01 Y100.;  <br>M99; | %@MACRO  <br>#1= SIN(100)G01 Y100.;  <br>M99; | 由于合并后指令不合法<br><br>会跳出COM-003<br><br>注：10.120.45 之后，将会跳出 COM-008 |
| %@MACRO  <br>G01 X100. // 漏加";"  <br>G01 Y100.;  <br>M99; | %@MACRO  <br>G01 X100.G01 Y100.;  <br>M99; | 由于合并后指令可直接执行<br><br>所以不会跳出警报<br><br>会直接走到X100. Y100.<br><br>注：10.120.45 之后，将会跳出 COM-008 |

使用到"( ... )"，刮号内的内容...，会视为注解省略。

使用到"(\* ... \*)"，刮号内的内容...，会视为注解省略。

依照 PR3201 车床程序所设定的操作习惯书写。

只能以车床习惯C-type书写。

备注：

1. 被当作副程序（子程序）或宏程序呼叫的加工档，建议不要使用多轴群档案（含轴群辨识符号$1、$2）。  
若有此应用情境，多轴群副程序档案必须小于60KB(60000bytes)，否则控制器将报警[COR-203 加工程序格式不符](https://syntecclub.atlassian.net/wiki/spaces/TechManual/pages/44123858/COR-203)；单轴群副程序则不在此限。
2. 不支援MACRO格式的多轴群（含轴群辨识符号$1、$2）的副程序。
3. 档案若超过60KB(60000bytes)
  1. 10.120.32及其以后的软件版本：支援IF、CASE、REPEAT、FOR、WHILE等等有范围区间的语法。
    1. 执行加工前建议执行图形模拟进行语法检查。
    2. 语法范围区间过大时，在加工时有可能会使轴向降速。可查看诊断变数D320「解译总线单节数」及D376「运动规划减速次数」来判断是否为语法范围区间过大造成轴向降速。**代码区块**
```
%@MACRO
G01 X30. Y30.;
#1 := 1;
IF #1 = 2 THEN
// ===================
// IF条件不成立
// 此处加工程序不执行
// ===================
END_IF;
G01 X100. Y100.;	// 执行此行前，轴向可能产生降速
```
  2. 10.120.32之前的软件版本：无法支援IF、CASE、REPEAT、FOR、WHILE等等有范围区间的语法，否则会报警[COR-204 档案太大](https://syntecclub.atlassian.net/wiki/spaces/TechManual/pages/44123860/COR-204)。
4. 除了程序注解、使用字串做为引数的Macro语法以外，仅允许以ASCII撰写加工程序。若使用者在加工程序中撰写了ASCII以外的字符，将会跳警报[COM-027 无效的字符](https://syntecclub.atlassian.net/wiki/spaces/TechManual/pages/44124108/COM-027)。
5. 支援 CR ("\\r")、LF ("\\n")、CR LF ( "\\r\\n" ) 三种换行方式。

# **指令格式Block Format**

  
单行动作控制指令的撰写格式叙述如下：

|     |     |     |     |     |     |     |     |     |     |     |     |     |     |     |     |     |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| /   | N   | G   | X   | Y   | Z   | A   | B   | C   | I   | J   | K   | F   | S   | T   | D   | M   |

|     |     |
| --- | --- |
| /   | 单节选择性跳跃功能，需配合PLC的C41；不支援函数语法及变数运算 |
| N   | 单节次序码，必须撰写在该单节的第一码位置，且不可在同一行后面撰写MACRO指令或变数指定 |
| G   | 功能指定码，需撰写在N码之后 |
| X   | X轴的移动命令，或是扩充G码的引数，需撰写在G码后 |
| Y   | Y轴的移动命令，或是扩充G码的引数，需撰写在G码后 |
| Z   | Z轴的移动命令，或是扩充G码的引数，需撰写在G码后 |
| A   | A轴的移动命令，或是扩充G码的引数，需撰写在G码后 |
| B   | B轴的移动命令，或是扩充G码的引数，需撰写在G码后 |
| C   | C轴的移动命令，或是扩充G码的引数，需撰写在G码后 |
| I   | X方向的的半径命令，或是扩充G码的引数，需撰写在G码后 |
| J   | Y方向的的半径命令，或是扩充G码的引数，需撰写在G码后 |
| K   | Z方向的的半径命令，或是扩充G码的引数，需撰写在G码后 |
| F   | 单节进给速度，或是扩充G码的引数 |
| S   | 主轴旋转速度，或是扩充G码的引数 |
| T   | 刀具选择功能，或是扩充G码的引数 |
| D   | 刀具补偿功能，或是扩充G码的引数 |
| M   | 辅助功能，或是扩充G码的引数 |

  
核心解译处理顺序（1.最先～10.最后）：

1. 模态G码（G15、G17、G70等）、扩充G码宏程序（G73、G84等）
2. M码宏程序、T码宏程序
3. S码
4. F码
5. H码
6. D码
7. T码
8. M码
9. B码
10. 插值G码（G0、G1等）、功能G码（G4、G51、G68等） 

备注：

1. 其余未说明之指令格式由相关G码以引数形式带入
2. 一般在副程序中会使用"GETARG"函数来读取引数内容，而在主程序（父程序）中可使用之引数形式规则如下：
  1. 使用引数D、E、H、I、J、K、L、M、P、Q、R、T，仅能使用单一符号传入引数，例如「G101 X30. Y40. D50. ;」，若在其后附带数字将引发警报，例如「G101 X30. Y40. D1=50. ;」
  2. 使用引数A、B、C、F、S、U、V、W、X、Y、Z，除了使用单一符号传入引数，亦可在其后附带数字，例如「G101 X30. Y40. Z1=50. ;」
  3. 使用引数时，若在同一单节下复数个相同的引数，解译时会使用最后一个引数值来执行。  
例如:
```
// 范例一，引数直接为轴向情形
G00 X10. X100.;  // 此时实际执行G00 X100.
G01 X20. X-100.; // 此时实际执行G01 X-100.
// 范例二，引数会被带入G码或Macro
G02 X60. C50. I-30 I20 J60 // 此时实际执行G02 X60. C50 I20 J60
G100 A10. B20. B50. C20.;  // 此时实际执行G100 A10. B50. C20.
// 范例三，功能G码
G10 L1000 L1500 P1 P2 Q1; // 此时实际执行G10 L1500 P2 Q1
```
  4. 使用引数亦可在其后进行数值或存值为数值之变数运算，运算式的括号可省略。  
例如:
```
#1 := 3.0;
G01 X(1.0+3.0) // 数值运算，执行的结果为G01 X4.0
G01 X(4.0*#1)  // 存值为数值之变数运算，执行的结果为G01 X12.0
G01 X30.0/2.0  // 运算式的括号可省略，执行的结果为G01 X15.0
```
  5. 以上所有动作指令之后，仅可使用数值或存值为数值之变数，否则可能因程序解译之编码限制而造成系统错误，此误用情况不在控制器的保护范围内。

# **运算子（Operator）**

|     |     |     |
| --- | --- | --- |
| 运算子 | 符号  | 执行顺序 |
| 括号  | ( ) \[ \] | 1   |
| 函数赋值 | Identifier (引数) | 2   |
| 负号  | \-  | 3   |
| 补数  | NOT | 3   |
| 乘号  | \*  | 4   |
| 除号  | /   | 4   |
| 模数（余数） | MOD | 4   |
| 加号  | +   | 5   |
| 减号  | \-  | 5   |
| 比较  | <,>,<=,>= | 6   |
| 等于  | \=  | 7   |
| 不等于 | <>  | 8   |
| 布林运算"且" | &AND, | 9   |
| 布林运算"互斥" | XOR | 10  |
| 布林运算"或" | OR  | 11  |

注1：

使用『/』元件（除号）时，需注意若分子与分母都是整数，所得结果仍为整数。整数及非整数之差别在有无加入小数点。

范例：

- 分子为非整数：1. / 2 = 0.5
- 分母为非整数：1/ 2.0 = 0.5
- 分子、分母均为整数：1/2 = 0
- 刮号内均为整数：(1/2)\*1.0 = 0

注2：

MOD运算子（模数）仅适用于数值型态为Long，若数值型态为Double，则会出现下述警报讯息。

范例：

%@MACRO  
@1:= 4. MOD 3;  
M99;

# **MACRO撰写注意事项**

- MACRO内建议多使用区域变数（Local Variables，#1 ~ #400），除非有跨MACRO的需求才使用公用变数（Global Variables，@1 ~ @165535）。
- 执行MACRO时，使用者的资料是透过引数（A\_、B\_、…,、Z\_、X1=、*Y1=、*…）传入，引数与区域变数相通，下表为引数与区域变数之关系。
- 针对扩充之引数位址，例如X1引数，请使用GETARG函数来读取其数值。
|     |     |     |     |     |     |
| --- | --- | --- | --- | --- | --- |
| 引数  | 对应变数 | 引数  | 对应变数 | 引数  | 对应变数 |
| A   | #1  | J   | #5  | T   | #20 |
| B   | #2  | K   | #6  | U   | #21 |
| C   | #3  | L   | #12 | V   | #22 |
| D   | #7  | M   | #13 | W   | #23 |
| E   | #8  | P   | #16 | X   | #24 |
| F   | #9  | Q   | #17 | Y   | #25 |
| H   | #11 | R   | #18 | Z   | #26 |
| I   | #4  | S   | #19 | X1  | GETARG(X1) |
- 模式变数（Modal Variables，#2001 ~ #2100、#3001 ~ #3100）在系统复位时，会回复成VACANT状态，因此可应用于多个MACRO间进行资料交换之时机，以节省变数资源之使用。
- MACRO若需要内定初始值，可多加利用使用者参数（Customer Parameter，#4001 ~ #4100、#5001 ~ #5100）。
- 执行MACRO副程序（子程序）时，若模式G码会被改变（G91/G90、G40/G41/G42,…等），请在一进入MACRO时，即备份当前的状态，待离开MACRO前再回复原有状态。
- 若想在离开MACRO后，继续保留此MACRO之插值模式（#1000），建议在离开MACRO前，将#1000指定为该MACRO之号码。尔后只要是轴向位移之指令单节，系统将自动呼叫此MACRO，而不用再次指定。
  - 插值模式在遇到G00/G01/G02/G03/G31/G33或#1000内容变动时，将被自动改写。
- 对于长度或角度的引数，在运算前请使用STD函数将单位标准化，以符合工具机使用习惯。
- 不可改变座标系统设定，像是G92/G54/G52等与座标系统相关之指令，否则图形模拟功能将失去参考意义。
- 执行加工时，核心会预解MACRO内容，因此MACRO执行速度会超前G/M码指令，若变数指定或资料读取需与G/M码动作同步，请于变数指定或资料读取前加入WAIT函数，以确保动作正确。  
注意：WAIT 函数目的是为了确保在运动指令执行完毕前，就因预解而提早读写相关数值所造成的错误，因此，其只确保 WAIT 前的 G/M 码执行完毕前不会继续往后预解 ( 其中 M98、M99、M198 三者为例外 )，对其他指令则不会有作用。
- MACRO程序最后需加"M99;"，才能返回主程序（父程序）。
- 请养成良好习惯，在程序中多加入注解，增加程序可读性，帮助后续维护及问题排除。

## 登录G码宏程序

- 开发人员可以根据机台需求来新增标准G码以外的G码宏程序，也可以客制标准G码的内容。
- 透过【Pr3701~3710 登录G码呼叫宏程序】设定，登录想要客制的标准G码，当程序中执行到对应的G码时，将不再执行标准G码之动作，而是执行客制G码宏程序之内容。
- 以下是【Pr3701~3710 登录G码呼叫宏程序】设定值与目前开放客制的标准G码对照表

| Pr3701 | 标准G码 | G码宏程序档案名称 |

| \-1 | G00 | G0000 |
| 1   | G01 | G0001 |
| 2   | G02 | G0002 |
| 3   | G03 | G0003 |
| 4   | G53 | G0053 |
| 5   | G40 | G0040 |
| 6   | G41 | G0041 |
| 7   | G42 | G0042 |
| 8   | G43 | G0043 |
| 9   | G44 | G0044 |
| 10  | G49 | G0049 |
| 11  | G04 | G0004 |

- 以下是登录G码宏程序之运作规格
  - 宏程序特性视同G码宏程序特性
  - 登录G码宏程序程序内的所有G码，均是标准G码
  - 标准G码中为插值模式的G码（例如G00、G01、G02、G03）在登录客制G码后，仍然具有继承功能  
    - 例如执行

Y100.  
其中Y100.也会执行G00宏程序，并且会读取占用Y引数
  - 若登录宏程序中，有改变插值模式( #1000 )，则务必在离开登录宏程序前，将插值模式还原
    - 例如登录G00为登录G码宏程序  
在G0000宏程序中，如果将插值状态变更为G01  
则在离开G0000前，需将插值模式变更回G00  
避免离开登录宏程序之后的状态错乱
  - 登录G码宏程序遇到以下指令时，将无法运作
    - 车床 G7.1
    - 车床 G12.1
    - 车床 ,A
    - 车床 ,R
    - 车床 ,C
    - 车床 所有加工循环指令
    - 铣床 所有加工循环指令
    - T码宏程序

# **变数规格**

- 有关#及@的变数规格说明，请参考[Macro变数规格](https://syntecclub.atlassian.net/wiki/spaces/TechManual/pages/44106246/Macro)
- 将不同型态的变数(Long和Double)一起做四则运算，会强制转型为Double再做计算，例如
```
@1 := 1;		// 以Long型态指派变数 @1
@2 := 2.0;		// 以Double型态指派变数 @2
@3 := @1 + @2;	// 得到结果为"@3 := 3.0"型态为Double
@4 := 2;		// 以Long型态指派变数 @4
@5 := 100;	 	// 以Long型态指派变数 @5
@6 := @4 * @5;	// 得到结果为"@6 := 200"型态仍为Long
```

## 宏程序使用说明

### 引言

- 控制器内建的基本G、T、M码可能无法满足各产业应用之需求，故新代科技提供客制宏程序的功能。  
开发人员可以根据其机台之特性及需求，开发对应的宏程序程式，提供机台专用之特殊动作，提升机台价值。
- 在开始介绍前，先针对主程式中，呼叫其他程式的方式进行定义：  
  - 呼叫副程式：执行呼叫之程式，不可读取、占用引数。
  - 呼叫宏程序：执行呼叫之程式，可以读取、占用引数。
  - 引数的定义请参阅[引数说明](https://syntecclub.atlassian.net/wiki/pages/viewpage.action?pageId=44107841)
- 後续章节将介绍宏程序程式的相关规格，呼叫副程式的规格将不赘述。

### 宏程序分类

- 所有宏程序均需要符合以下条件
  - 撰写语法须符合宏程序语法
    - 程序内容以%@MACRO开头
    - 每行以分号"；"结尾
  - 档名结尾不得有副档名
  - 程序内容结尾必须是M99，回到呼叫的主程序
- 宏程序基本上可以分以下几类
  - G码宏程序
  - 非模态调用宏程序G码（G65）
  - 模态调用宏程序G码（G66/G66.1）
  - T码宏程序
  - M码宏程序

### 宏程序运作流程说明

- 以下主要以G码宏程序作为范例说明，其他宏程序若有规格不同之处，会特别说明。
- 当加工程序执行到G码时，会执行G码宏程序的内容。
- G码宏程序内最後一行撰写M99，当G码宏程序执行到M99之後，会回到主程式继续往下加工。
- G码宏程序内的动作可能会改变系统状态，例如在G码宏程序内下G90/G91，会使命令模式改变。
  - 为了避免影响到主程式的执行，或者其他宏程序/副程式的运作，通常希望主程式的状态，在执行G码宏程序的前後状态一致。
  - 所以建议G码宏程序内容的一开始要备份系统状态，G码宏程序的结尾要将系统状态还原。
-  范例
  - 主程式中会下G200这个指令，进而执行G0200这个G码宏程序
  - G0200中会先备份系统状态
  - 接着使X轴增量移动10.
  - 在离开前将状态还原

**范例\_Main**

```
// Main
G90;
G01 X10. F100.;          // X轴透过G01移动    执行结果X=10.
G200;                    // X轴透过G200移动   执行结果X=20.
X-20.;                   // X轴透过G01移动    执行结果X=-20.
M30;
```

**范例\_G0200**

```
// G0200
%@MACRO
#101 := #1000;                  // 备份#1000状态，G00/G01/G02/G03/G33/G34/G35
#102 := #1004;                  // 备份#1004状态，G90/G91
G91 G00 X10.;                   // X轴透过G00增量移动10.   执行结果X=20.;
G#101;                          // 还原#1000状态，G00/G01/G02/G03/G33/G34/G35
G#102;                          // 还原#1004状态，G90/G91
M99;                            // 回到主程式
```

### 引数用途简介

- 宏程序可以执行开发人员设计的动作，从简单的状态转换，到复杂的多工序动作。
- 若一个宏程序只能执行一个流程，那开发人员为了各种状况就需要撰写数不清的宏程序。
- 比如某一个宏程序的功能是切削出10公分的正方形，这时若想切削20公分的正方形，就需要再撰写一个新的宏程序。
- 这样的宏程序太过呆板，此时可以透过引数，对宏程序的执行内容进行变数调整，例如正方形的边长，让该宏程序具有更高的弹性。
- 范例
  - 主程式中为了切削出10公分的正方形，所以下了G200
  - 主程式中为了切削出20公分的正方形，所以下了G201
  - 主程式中为了切削出任意大小的正方形，所以下了G202，大小边长由G202的引数X来决定。

// Example002_Main
G90 G00 X0. Y0.;
G200;
G201;
G202 X30.;
M30;
```

| --- | --- | --- | --- | --- |
| 分类  | 轴向引数 | 条件引数 | 特殊引数 | 例外引数 |

- 引数会被宏程序读取占用。
- 「读取」表示该宏程序内有撰写到该引数对应的#变数，故该宏程序读取该引数进行运算。
- 「占用」表示该引数被读取后，无法再被其他宏程序读取，称之为引数该宏程序占用。
- 引数被读取之后不代表就会被占用，要看读取的宏程序特性。请参阅后续章节【宏程序解译处理顺序】【宏程序特性】。
- 引数没有被读取不代表不会被占用，要看引数的特性。请参阅此章节后续说明。
- 同一行内有重复的引数，并不会发生警报，但只有最后一个引数的数值会被读取。
- 可以理解为重复的引数均是对同一个#变数填值，后面的引数数值盖掉前面的引数数值。
- 引数基本上可以分为以下几类

### 宏程序解译处理顺序

- 在说明宏程序读取占用引数之前，先说明宏程序解译处理顺序。
- 加工程序会从第一行开始依序往下解译，如果每一行都只撰写一个宏程序/指令，则不会发生引数抢占的问题。
- 当同一行内下达了多种宏程序或指令，系统就会依照解译处理顺序来处理引数，以下说明「解译处理顺序」。

| 解译顺序 | 类型  | 细项  | 举例  |
| --- | --- | --- | --- |
| 1   | 部分G码 | G码宏程序 | G73、G84等 |
| 模态G码 | G15、G17、G70等 |
| 非模态调用宏程序G码 | G65 |
| 以上各项根据撰写顺序（由左至右），先解译者先取得引数。 |     |
| 2   | 宏程序 | M码宏程序 |     |
| T码宏程序 |     |
| 以上各项根据撰写顺序（由左至右），先解译者先取得引数。 |     |
| 3   | S码  |     |     |
| 4   | F码  |     |     |
| 5   | H码  |     |     |
| 6   | D码  |     |     |
| 7   | T码  |     |     |
| 8   | M码  |     |     |
| 9   | B码  |     |     |
| 10  | 功能G码 |     | G04、G51、G68等 |
| 11  | 插值G码 |     | G00、G01等 |

  

- 通常解译处理顺序同时也代表着引数占用的顺序，表示「先解译处理的宏程序，优先读取占用引数」；  
但并非绝对成立，还要根据该宏程序之特性而定(部分宏程序有先解译后执行的特性，将不满足先解译先占用的规则)。
- 举例来说：  
1\. 功能G码和插值G码有在解译处理顺序上有先后，因此功能G码会先于插值G码占用轴向引数；  
　无论撰写的先后，在同一单节执行时，功能G码会优先占用轴向引数。  
2. 从解译处理顺序中可以发现，M码宏程序及T码宏程序两者的顺序等级都是Lv2，  
　所以两者如果写在同一行，解译处理的顺序是根据撰写的先後，写在前面的会先被解译处理。

### 宏程序呼叫范例

- 前述的宏程序特性提到诸多宏程序的使用状况，以下将针对宏程序呼叫引数的各种状况，提供对应的说明及范例。
- 没有特别提到是哪一种宏程序的范例，表示所有宏程序均适用。  
  

#### 01\_引数与程序变数（区域变数）

```

#### 02\_引数重复撰写

- 重复的引数只有最後一个引数会被读取
- 可以理解为重复的引数均是对同一个#变数填值，後面的引数数值会盖掉前面的引数数值。
**Main**
```
// Main
G01 X0.;			
G200 X10. X-10.;	// X引数写了两次，只有X-10.会被G200读取
					// 可以视为#24先被X10.填成10.後
					// 又被X-10.填成-10.
					// G200执行完毕後，执行G01
					// 因为此行的引数均已被占用
					// 所以G01读取不到任何引数
M30;
```
**G0200**
```
// G0200
%@MACRO
#100 := #1000;		// 备份插值模式
#101 := #24;		// 读取到的#24是-10.
G01 X#101;			// 移动到X-10. 
M00;
WAIT();
G#100;				// 还原插值模式
M99;
```

#### 03\_轴向引数与条件引数

- 轴向引数只要其中一个被占用，则其他轴向引数都会一起被占用。
  - 若不确定G码的执行情况，且希望轴向引数都会被占用，避免未占用时被G01继承而产生不预期的轴向移动，则建议在G码内部先用变数存取所有引数，再使用，确保所有引数被占用。
- 条件引数各自独立，不会互相影响。
**Main**
```
// Main
G01 X0. Y0. Z0.;
G200 X10. Y20. Z30.;  				// 虽然G200只有读取X引数
									// 但Y、Z引数也一起被占用
									// G200执行完毕后，会执行G01
									// 因为此行的引数均已被占用
									// 所以G01读取不到任何引数
G01 X0. Y0. Z0. F100.;  
G201 P20. X10. Y20. Z30. F200.;		// G201只有读取P引数
									// 所以XYZF引数都还没有被占用
									// G201执行完毕后，执行G01
									// G01会读取到X10. Y20. Z30. F200.
									// 并执行对应动作
M30;
```
**G0200**
```
// G0200
%@MACRO
@1:=#24;			// 只有读取X引数
M00;
WAIT();
M99;
```
**G0201**
```
// G0201
%@MACRO
@1:=#16;			// 只有读取P引数
M00;
WAIT();
M99;
```

#### 08\_多个G码宏程序在同一行

- 当多个G码撰写在同一行时，不管是G码指令还是G码宏程序，均会依序对每一个G码宏程序进行解译处理。
- 但只有写在最後的G码宏程序可以读取及占用引数，前面的G码虽然会先解译处理，但不会读取任何引数，因此也不会占用任何引数。

							// G201读取占用P引数後，执行G01
							// G01会读取占用X10.
							// 并执行对应动作
M30;
```

#### 09\_G码指令与G码宏程序在同一行

- 当多个G码撰写在同一行时，不管是G码指令还是G码宏程序，均会依序对每一个G码宏程序进行解译处理。
- 但只有写在最後的G码宏程序可以读取及占用引数，前面的G码虽然会先解译处理，但不会读取任何引数，因此也不会占用任何引数。

**Main**

```
// Main
G90;
G00 X0. Y0. Z0.;			// G00 定位到X0 Y0 Z0
G200 G01 X10. F100.;		// 因为两个G码写在同一行
							// 故只有最後一个G码可以读取引数
							// G200会执行，但读取不到任何引数
							// G200执行完毕後，执行G01 X10. F100.;
							
G00 X0. Y0. Z0.;			// G00 定位到X0 Y0 Z0
G01 G201 X10. F100.;		// 因为两个G码写在同一行
							// G01先解译，此时系统状态变更为G01
							// 接着解译G201
							// G201内部执行G01 X10. F100.
							// 因为G201有读取占用轴向引数，所以所有轴向引数均已被占用
							// 接着G01发现没有任何後续移动指令，所以没有执行任何动作
 
G00 X0. Y0. Z0.;			// G00 定位到X0 Y0 Z0
G01 G202 X10. F100.;		// 因为两个G码写在同一行
							// G01先解译，此时系统状态变更为G01
							// 接着解译G202
							// G202内部没有执行任何动作，也没有读取占用任何引数
							// 接着G01发现轴向引数尚未被占用，所以执行G01 X10. F100.
M30;
```

#### 11\_G码宏程序与T码宏程序在同一行

- 【Pr3215 选刀时呼叫模式】设定为2，T0000为T码宏程序。
- 当G码宏程序与T码宏程序在同一行时，会先解译处理G码宏程序，再解译处理T码宏程序。
- 如果G码宏程序占用了T码，则T码宏程序将不执行。
- 如果G码宏程序不占用T码，则T码宏程序会在G码宏程序执行完毕後执行。
- 系统中只会有一个T0000，不过以下范例会以T0000\_T01和T0000\_T02来表示前後两次T码宏程序。

**Main**

```
// Main
G01 X0. Y0. Z0.;
T01 G200 X10. Y20. Z30. F1000;		// 解译顺序是先解译G码宏程序
									// G码宏程序（G0200）内有读取T码引数
									// T码引数视为被占用，所以不会执行T码宏程序
									// G码宏程序（G0200）执行完毕後，执行G01
									// G01会读取占用X10. Y20. Z30. F1000
									// 并执行对应动作
 
T02 G201 X10. Y20. Z30. F1000.;		// 解译顺序是先解译G码宏程序
									// G码宏程序（G0201）内没有读取T码引数
									// T码引数视为尚未被占用
									// 所以G码宏程序（G0201）执行完毕後，会再执行T码宏程序
									// G码宏程序（G0201）虽然有占用X引数
									// 但是T码宏程序会读取所有想要读取的引数，所以还是读取的到XYZF
									// 执行完T码宏程序後，会再执行G01
									// 因为此行的引数均已被占用
									// 所以G01读取不到任何引数
M30;
```

#### 14\_M码宏程序

- ```
【Pr3601~3610 登录M码呼叫宏程序】设定为123，登录M0123为M码宏程序。
```
- M码被登录为M码宏程序时，需要在M码没有被占用，同时也没有任何轴向引数被占用时，才会执行M码宏程序。
- M码宏程序执行时会占用除了T码以外的所有引数。
- 不管M码有没有被当做M码宏程序使用，M码引数（#13）都会被更新。
- 只有M码被当做辅助码时，M码对应的R值才会被更新。M码被当做宏程序或引数时，M码对应的R值均不会更新。

**Main**

```
// Main
G01 X0. Y0. Z0. F100.;
M123 X10. Y20. Z30. F300.;  // M码引数没有被占用
							// 也没有任何轴向引数被占用
							// 所以会执行M码宏程序M0123
							// 因为M码宏程序执行後，除了会占用M码宏程序已读取的引数之外，还会占用除了T码以外的所有引数
							// F引数虽然没有被M0123读取，但会一起被视为占用，所以进给速度不会被更新
							// M123执行完毕後，执行G01
							// 因为此行的引数均已被占用
							// 所以G01读取不到任何引数
							
G01 X0. Y0. Z0.;			// 此行的F速度是F100. 因为上一行的F引数被M码宏程序占用
M30;
```

**M0123**

```
// M0123
%@MACRO
#100 := #1000;				// 备份插值模式
#101 := #24;				// 读取X引数

## MACRO 支援字符

- 可显示字符
  - 数字：0-9
  - 英文字母
    - A-Z
    - a-z
  - 其他（半形）
