# SYNTEC 宏程序 VSCode 扩展

为新代（Syntec）CNC 宏程序提供语法高亮、智能补全、悬停文档、G/M代码跳转、实时语法检查。

## 功能

| 功能 | 说明 |
|------|------|
| **语法高亮** | %@MACRO、控制流、60+函数、G/M代码、变量、字符串 |
| **智能补全** | 输入函数名 → 自动弹出含签名参数的补全列表 |
| **Hover 文档** | 悬停函数名 → 显示完整说明、参数解释、使用示例 |
| **GOTO 跳转** | Ctrl+点击 N标签 → 跳转到定义行；G65 Pxxx → 跳转宏程序 |
| **实时诊断** | IF/END_IF/FOR/WHILE/CASE 块配对、括号匹配、中文字符检测 |
| **Outline 大纲** | N标签 → VSCode 大纲/符号导航 |
| **代码片段** | 40+ 模板（IF/FOR/DB/IO/报警等） |

## 支持的文件类型

- `.G` `.nc` `.macro` `.scp`
- 无扩展名且首行为 `%@MACRO` 的文件

## 已知限制

- GOTO 跳转依赖工作区中存在目标宏程序文件
- 诊断仅做块匹配和字符检查，不做完整语义分析
- 暂不支持宏变量重命名、查找引用等深度分析

### 控制器实测确认的语法规则

| 规则 | 说明 |
|------|------|
| GOTO | `GOTO 100;`（不带N），目标 `N100;`（必须分号结尾） |
| ELSIF | 不支持，请使用 `ELSEIF` |
| INT() | 不支持，请使用 `FLOOR()` 或 `CEIL()` |
| [] | 不能用于运算，仅限间接定值 `@[#变量]` |
| NOT | 只能对整数使用 |
| XOR | 支持 `IF #1 XOR #2` 和 `IF (1=1) XOR (1=1)` 两种形式 |
